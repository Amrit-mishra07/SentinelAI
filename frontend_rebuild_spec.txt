```
--color-accent:         #4f8ef7   /* primary blue — links, active states, primary CTA */

--color-critical:       #f85149   /* severity: critical */
```

**The signature element of this design:** a 1px `border-left` on every panel card using a 
gradient that goes from the severity color at the top to transparent at the bottom. Critical 
panels have a red left-border gradient. This is the visual motif that ties every card 
together and makes the UI feel engineered rather than designed.

### Typography

Install via `next/font/google`:
- **Geist** as the primary sans-serif (Vercel's typeface — clean, technical, modern)
- **Geist Mono** as the monospace (file paths, code, hashes, IDs, scan output)

If Geist is not available in next/font/google, fall back to **Inter** + **JetBrains Mono**.

Type scale:
```
10px / weight 500 / tracking 0.08em  → EYEBROW LABELS (e.g., "CRITICAL", "REPOSITORIES")
12px / weight 400                    → table cells, meta, timestamps  
13px / weight 400                    → body text, descriptions
14px / weight 500                    → nav items, button text, table headers
16px / weight 500                    → card titles, section headings
20px / weight 500                    → page headings
24px / weight 600                    → metric card numbers (medium)
36px / weight 600 / tabular nums     → hero stat numbers (large)
```

Monospace appears on: file paths, commit hashes, scan IDs, line numbers, code diffs,
API endpoints, repository URLs. Always use `font-mono` for these. Never render a SHA or
file path in the sans-serif face.

### Spacing + Layout

- Sidebar: `220px` fixed left, full viewport height, `position: fixed`
- Topbar: `56px` height, `position: sticky top-0`, `z-index: 40`, backdrop blur
- Content area: `margin-left: 220px`, `padding: 24px 32px`, scrollable
- Card border-radius: `8px` 
- Input border-radius: `6px`
- Button border-radius: `6px`
- Gap between grid items: `16px`
- Section spacing: `32px`

## `lib/api-client.ts` — rebuild this completely

Requirements:
- Generic typed methods: `get<T>`, `post<T>`, `put<T>`, `delete<T>`
- Reads token from localStorage, guards SSR with `typeof window !== 'undefined'`
- Throws typed `ApiError` (with `status` + `message`) on non-2xx
- On 401: dispatch a custom DOM event `'sentinelai:unauthorized'` (not a direct redirect
  — let the auth hook handle navigation to avoid React state issues)
- 30 second timeout via `AbortController` — throw a specific `RequestTimeoutError`
- In development: log every request and response to console with timing
- Never swallow errors silently

```typescript
// Shape of the client
export const api = {
  get: <T>(path: string) => Promise<T>
  post: <T>(path: string, body?: unknown) => Promise<T>
  put: <T>(path: string, body?: unknown) => Promise<T>
  delete: <T>(path: string) => Promise<T>
}
```

## `lib/formatters.ts`

Implement every one:

```typescript
formatDate(iso: string): string
  // "Jun 24, 2026"

formatTime(iso: string): string
  // "14:32 UTC"

formatRelative(iso: string): string
  // "just now" | "2 minutes ago" | "3 hours ago" | "yesterday" | "Jun 24"
  // switch to absolute date when > 7 days ago

formatDuration(startIso: string, endIso: string | null): string
  // "1m 23s" | "45s" | "—" when endIso is null

formatFilePathTruncated(path: string, maxLen?: number): string
  // truncates the MIDDLE: "src/.../auth/queries.py:42"
  // never truncate the filename itself, only intermediate directories

formatCount(n: number): string
  // < 1000: "742"
  // >= 1000: "1.2k"
  // >= 1_000_000: "1.2M"

pluralize(n: number, singular: string, plural?: string): string
  // pluralize(1, 'scan') → '1 scan'
  // pluralize(3, 'scan') → '3 scans'

computeRiskScore(counts: SeverityCounts): number
  // critical×10 + high×7 + medium×4 + low×1
  // clamp to 0–100

  // "3f4a8b2c-..." → "#3f4a8b"
```

Score computed as: `min(100, critical×10 + high×7 + medium×4 + low×1)`

**Full vulnerability table** (same as scan detail but with AI analysis column always visible).

**Download report** button: calls `POST /api/report/[scanId]/download`. On success: toast
"Report download started". On error: toast with error message.

- Hide sidebar, topbar, action buttons

**`Toast` + `ToastProvider`**

  `toast.info(msg)`

## Hooks

**`usePolling(fn, intervalMs, shouldPoll)`**
```typescript
// Generic polling hook
// shouldPoll: boolean — stop polling when false
// Clears interval on unmount automatically
// Does not poll when document is hidden (uses visibilitychange event)
function usePolling(
  fn: () => Promise<void>,
  intervalMs: number,
  shouldPoll: boolean
): void
```

**`useAuth()`**
- Listens for `sentinelai:unauthorized` event (dispatched by api-client)
- On that event: clear token, redirect to `/login` with toast "Session expired"
- Returns `{ user, token, logout, loading }`
- `logout()`: DELETE token, redirect to login, clear any cached data
- Guards SSR: return `{ loading: true }` until after hydration

**`useCountUp(target, duration)`**
- Animates a number from 0 to `target` over `duration` ms
- Uses `requestAnimationFrame`
- Returns current animated value as a number
- Used in MetricCard to animate the big number on load

**`useKeyboard()`**
Global keyboard shortcut handler. Register these shortcuts:
- `⌘K` / `Ctrl+K` → open search modal
- `N` → open new scan modal (when not in an input)
- `Escape` → close any open modal

## Performance requirements

- No chart library dependency — all charts are pure SVG (runtime is ~0 overhead)
- `usePolling` pauses when `document.hidden` is true (tab not visible)
- Metric card `useCountUp` respects `prefers-reduced-motion` — skip animation if set
- Large vulnerability lists (>100 items) should use windowed rendering or pagination
- Modal renders lazily (not in DOM until opened)
- API responses should be cached in component state — don't re-fetch when navigating
  back to a previously loaded page within the same session
